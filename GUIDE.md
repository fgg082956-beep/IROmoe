## Как теперь считаются баллы

### Технические индикаторы (старая часть, −10..+10)

| Индикатор          | Макс. вклад | Условие                                              |
|--------------------|-------------|------------------------------------------------------|
| RSI                | ±2          | < 30 → +2; < 40 → +1; > 70 → −2; > 60 → −1         |
| MACD               | ±2          | бычье пересечение → +2; выше сигнальной → +1 (и обратно) |
| Скользящие MA      | ±1          | цена выше MA5 > MA10 > MA20 → +1                    |
| Bollinger Bands    | ±2          | цена ниже нижней → +2; у нижней → +1                |
| Объём              | ±1          | аномальный объём при росте → +1                     |
| Стохастик          | ±2          | бычье пересечение < 20 → +2; просто < 20 → +1       |
| **Итого техника**  | **−10..+10**|                                                      |

### Новостной сентимент (новая часть, −3..+3)

Каждый запуск `main.py` перед теханализом:
1. Собирает до 10 свежих заголовков на каждый тикер из Google News + MOEX RSS
2. Каждый заголовок прогоняет через модель `rubert-tiny2-russian-financial-sentiment`
3. Модель возвращает `positive (+1)`, `negative (−1)` или `neutral (0)`
4. Все баллы суммируются → получается сырой новостной балл
5. Балл **ограничивается диапазоном ±NEWS_SCORE_CAP** (по умолчанию ±3 в config.py)

Это ограничение (`CAP`) сделано специально, чтобы новости не перевешивали технический анализ.

### Итоговый балл и смещение нуля

```
итого = технический балл + новостной балл
```

**Максимальный диапазон при настройках по умолчанию (CAP=3):**

```
мин: -10 + (-3) = -13
макс: +10 + (+3) = +13
нейтральная зона: от -3 до +3 (симметрична)
```

**Почему ноль «смещается»?**
Раньше чистый нейтраль = 0. Теперь у тикера с позитивным новостным фоном балл будет выше нуля даже при нейтральной технике. Это и есть смещение: одни и те же технические показатели дадут разный сигнал в зависимости от новостного фона — что правильно.

**Пороги сигналов (в config.py):**

```python
SCORE_LONG_THRESHOLD  = +3   # итого >= +3  → ЛОНГ
SCORE_SHORT_THRESHOLD = -3   # итого <= -3  → ШОРТ
```

Один позитивный заголовок даёт +1. Без технических подтверждений это не даст ЛОНГ (нужно ещё +2 от техники).

### Пример

```
SBER:
  RSI 29 (перепроданность)       → +2
  MACD: бычье пересечение        → +2
  Боллинджер: цена ниже нижней   → +2
  Объём: норма                   → 0
  Стохастик: нет сигнала         → 0
  MA: нет чёткого тренда         → 0
  ─────────────────────────────────
  Технический балл:              = +6

  Новости: 4 позитивных + 2 нейтральных = +4, ограничено cap: → +3
  ─────────────────────────────────
  ИТОГОВЫЙ БАЛЛ: +6 + +3 = +9  → ЛОНГ ✅
```

## Установка (новая часть — шаг 2.5)

После стандартного шага `pip install -r requirements.txt` добавились 4 новые библиотеки:

```
feedparser    — ~50 КБ, устанавливается мгновенно
transformers  — ~4 МБ, устанавливается мгновенно
torch         — ~700 МБ (CPU-версия!) — скачивается долго, один раз
schedule      — ~30 КБ, устанавливается мгновенно
```

**Torch — большой пакет.** Первая установка займёт 5–15 минут в зависимости от скорости интернета. Последующие запуски — мгновенные, всё в кэше.

Команда установки (та же, что и раньше):
```
pip install -r requirements.txt
```

### Первая загрузка модели сентимент-анализа

При первом запуске `main.py` (или `python news_sentiment.py --once`) программа скачает модель `rubert-tiny2-russian-financial-sentiment` (~29 МБ) с HuggingFace. Это происходит автоматически, один раз. После этого модель хранится в кэше (`~/.cache/huggingface/`) и не скачивается повторно.

### Проверка (без запуска основной программы)

```
python news_sentiment.py --once
```

Должны увидеть:
```
Запуск сбора новостей: 2026-03-28 12:00:00
[Сбербанк] Google News: найдено 8 заголовков
[SBER] Итог: балл=+2  (+4 / -2 / ~2)
...
Данные сохранены в news_sentiment_data.json
```

### Посмотреть последние баллы без запуска анализа

```
python news_sentiment.py --scores
```

## Настройка весов новостей (config.py)

```python
NEWS_SCORE_WEIGHT = 1.0   # 1.0 = один заголовок = один балл
                           # 0.5 = новости влияют вполовину меньше
NEWS_SCORE_CAP    = 3      # максимальный вклад новостей = ±3
                           # поставь 0, чтобы полностью отключить новости

SCORE_LONG_THRESHOLD  = 3  # порог ЛОНГ (при изменении CAP — пересчитай)
SCORE_SHORT_THRESHOLD = -3  # порог ШОРТ
```

**Как подобрать CAP и пороги:**
- При `NEWS_SCORE_CAP = 0` — поведение идентично старой версии
- При `NEWS_SCORE_CAP = 1` — новости могут дать лишь небольшой «тай-брейк»
- При `NEWS_SCORE_CAP = 3` (по умолчанию) — новости весомы, но техника важнее
- При `NEWS_SCORE_CAP = 10` — новости могут полностью перевесить технику (не рекомендуется)

## Как добавить свои тикеры

Откроешь `config.py` — там есть два места:

1. Список `TICKERS` — тикеры для теханализа (как раньше)
2. Словарь `TICKER_TO_COMPANY` — маппинг тикер → название компании для поиска новостей

Добавляй синхронно в оба:
```python
TICKERS = [..., "YDEX"]

TICKER_TO_COMPANY = {
    ...,
    "YDEX": "Яндекс",
}
```

## Устранение проблем

### torch не устанавливается
```
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

### Модель не скачивается (нет доступа к HuggingFace)
Используй VPN. Или скачай модель вручную с [huggingface.co/mxlcw/rubert-tiny2-russian-financial-sentiment](https://huggingface.co/mxlcw/rubert-tiny2-russian-financial-sentiment) и положи в `~/.cache/huggingface/`.

### Новости не собираются (нет заголовков)
Проверь, открывается ли в браузере:
```
https://news.google.com/rss/search?q=Сбербанк&hl=ru&gl=RU&ceid=RU:ru
```
Если нет — попробуй VPN.

### Баллы всегда 0 у новостей
Значит модуль `news_sentiment.py` не установлен или произошла ошибка при загрузке модели. Запусти:
```
python news_sentiment.py --once
```
и посмотри на вывод — там будет конкретная ошибка.
